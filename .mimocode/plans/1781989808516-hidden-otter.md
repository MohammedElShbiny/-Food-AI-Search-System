## Current State Analysis

**Why startup is slow (2 min):**
- `BackgroundScraper.__init__` (line 21 of main.py) runs at module import time
- `bg_scraper.start()` in lifespan fires immediately, scraping 50 foods from 10+ websites
- This hogs network/IO resources and makes the server appear unresponsive during startup
- The `ScraperPauseMiddleware` wraps every request but doesn't block startup

**Current load balancing:**
- Only works in `NODE_MODE=coordinator` — requires manual env config
- The coordinator already proxies search requests to workers via round-robin
- But users can't add workers from the dashboard

## Changes

### 1. Fast Startup — `main.py` lifespan + `bulk_scraper.py`
- **Don't auto-start scraper on server boot** — change `BACKGROUND_SCRAPER_ENABLED` default to `false`
- Add a small startup delay (2s) before scraper starts, so server responds first
- In `main.py` lifespan: start scraper as a background task AFTER yield point begins, not blocking startup

**File: `main.py`** (lifespan, ~line 30-44)
```python
@asynccontextmanager
async def lifespan(application):
    if coordinator:
        await coordinator.start_health_checks()
    if worker_node:
        await worker_node.register()
        await worker_node.start_heartbeat()
    # Start scraper as non-blocking background task
    if config.BACKGROUND_SCRAPER_ENABLED and config.NODE_MODE != "worker":
        asyncio.create_task(_start_scraper_delayed())
    yield
    await bg_scraper.stop()
    ...

async def _start_scraper_delayed():
    await asyncio.sleep(2)  # Let server bind first
    await bg_scraper.start()
```

**File: `config.py`** — Change default:

### 4. Auto Load Balancing in Standalone Mode — `main.py` search endpoints
Currently, load balancing only works in coordinator mode. To make it work without changing client URLs:

Add a `LINKED_SERVERS` config (or use a JSON file `linked_servers.json`) that stores device URLs. In standalone mode, if linked servers exist, the server distributes search requests to them as well.

**New approach:** Store linked servers in a JSON file (persistent across restarts). Add API endpoints:
- `GET /api/servers` — list linked servers
- `POST /api/servers` — add a linked server
- `DELETE /api/servers/{name}` — remove a linked server

Modify search endpoints to proxy to linked servers when available, even in standalone mode.

**File: `main.py`** — New linked server manager class:
```python
class LinkedServers:
    def __init__(self):
        self.file = "linked_servers.json"
        self.servers = self._load()
    
    def _load(self):
        if os.path.exists(self.file):
            with open(self.file) as f:
                return json.load(f)
        return {}
    
    def _save(self):
        with open(self.file, "w") as f:
            json.dump(self.servers, f, indent=2)
    
    def add(self, name, url):
        self.servers[name] = {"name": name, "url": url, "added_at": datetime.now().isoformat()}
        self._save()
    
    def remove(self, name):
        if name in self.servers:
            del self.servers[name]
            self._save()
            return True
        return False
    
    def get_all(self):
        return list(self.servers.values())
    
    async def check_health(self, url):
        async with httpx.AsyncClient(timeout=3) as client:
            try:
                r = await client.get(f"{url}/api/health")
                return r.status_code == 200
            except:
                return False
```

Modify search GET/POST to try linked servers first (round-robin), fall back to local DB.
