# Plan: Add AI API Fallback + Web Scraping to Food AI

## Goal
When a food search returns no results from the local SQLite database, fall back to:
1. **Web scraping** from Arabic (Webteb, Mawdoo3, Supermama) and English (USDA, Healthline, Nutritionix) websites
2. **AI API call** (OpenAI, Gemini, Claude, OpenRouter, DeepSeek, NVIDIA) if scraping fails
3. **Auto-save** any found results back to the database for future instant lookups

## New Files to Create

### `scraper.py` — Web Scraping Module
- Uses `httpx` (async) + `beautifulsoup4` for HTML parsing
- `FoodScraper` class with methods:
  - `scrape_arabic(query)` — searches Webteb, Mawdoo3, Supermama in sequence until carbs found
  - `scrape_english(query)` — searches USDA FoodData Central API, Healthline, Nutritionix
  - Each site gets a dedicated `_scrape_<site>(query)` method with site-specific CSS selectors/parsing
- Returns `Optional[Food]` (None if nothing found)
- Configurable timeout per site

### `ai_providers.py` — Multi-Provider AI Fallback
- `AIProvider` ABC with `async def search(query: str) -> Optional[Food]`
- Concrete implementations:
  - `OpenAIProvider(api_key)` — calls GPT with structured prompt asking for carbs
  - `GeminiProvider(api_key)` — calls Gemini API
  - `ClaudeProvider(api_key)` — calls Anthropic API
  - `OpenRouterProvider(api_key)` — calls OpenRouter (multi-model gateway)
  - `DeepSeekProvider(api_key)` — calls DeepSeek API
  - `NVIDIAProvider(api_key)` — calls NVIDIA NIM API
- `AIFallback` orchestrator: tries providers in priority order, returns first successful `Food`
- All use `httpx` async client; prompt asks AI to return JSON with `food_id`, `name_en`, `name_ar`, `carbs`, `category_en`, `category_ar`, `serving_description`

### `fallback.py` — Fallback Orchestrator
- `FallbackSearch` class combining scraper + AI providers
- Flow: `search(query)` → scrape Arabic → scrape English → AI providers → return `Optional[Food]`
- Handles language detection (reuse existing `is_arabic`)

## Search Flow (End to End)

```
User query → DB search
  ├─ Found → return results (source="database")
  └─ Not found → FallbackSearch
       ├─ Arabic websites (if Arabic query)
       │   ├─ Webteb → parse carbs → save to DB → return
       │   ├─ Mawdoo3 → parse carbs → save to DB → return
       │   └─ Supermama → parse carbs → save to DB → return
       ├─ English websites (if English query or Arabic sites failed)
       │   ├─ USDA FoodData Central API → save → return
       │   ├─ Healthline → parse carbs → save → return
       │   └─ Nutritionix → parse carbs → save → return
       └─ AI providers (if all scraping failed)
           ├─ OpenAI → structured prompt → parse JSON → save → return
           ├─ Gemini → ...
           ├─ Claude → ...
           ├─ OpenRouter → ...
           ├─ DeepSeek → ...
           └─ NVIDIA → ...
  └─ All fail → return "no results" message
```
