# Multi-Device, Frontend, Performance & Smart Scraping Plan

## Overview
Four major features:
1. Worker node system (other devices register via ngrok, main server distributes scraping)
2. React/Vite TypeScript frontend (database viewer, add food, scraper status)
3. Performance optimization (WAL mode, connection reuse, skip already-scraped)
4. Smart scraping priority (pause fan-out during user requests)

---

## Feature 1: Worker Node System

### Architecture
```
Main Server (port 8000)
  ├── API + Scraper
  ├── Worker Registry (in-memory dict)
  └── Task Distributor
        ├── POST /api/workers/register  → worker registers with ngrok URL
        ├── GET  /api/workers           → list active workers
        ├── POST /api/workers/scrape    → send scrape task to worker
        └── DELETE /api/workers/{id}    → unregister worker

Worker Node (same codebase, different device)
  ├── Registers with main server on startup
  ├── Receives scrape tasks from main server
  ├── Scrapes and returns results
  └── Sends results back to main server for DB storage
```

### Files to Create/Modify

| File | Changes |
|------|---------|
| `worker.py` | NEW - WorkerNode class that registers with main server, receives tasks, scrapes, returns results |
| `main.py` | Add `/api/workers/*` endpoints, add task distribution logic in BackgroundScraper |
| `config.py` | Add `WORKER_MODE`, `MAIN_SERVER_URL`, `WORKER_ID` settings |

### Worker Registration Flow
1. Worker starts, sends `POST {MAIN_SERVER_URL}/api/workers/register` with `{id, ngrok_url, capabilities}`
2. Main server stores worker in registry with last-seen timestamp
3. Main server health-checks workers periodically (GET /api/health)
4. If worker not seen for 60s, mark as offline

### Task Distribution
1. BackgroundScraper checks available workers before each batch
2. If workers available, distributes food items across them via `POST {worker_url}/api/worker/scrape`
3. Workers scrape independently, return results
4. Main server stores all results in shared SQLite DB

## Feature 3: Performance Optimization

### SQLite WAL Mode
In `database/sqlite_db.py`, enable WAL mode on connection:
```python
def _get_conn(self):
    conn = sqlite3.connect(self.db_path)
    conn.execute("PRAGMA journal_mode=WAL")
    conn.execute("PRAGMA busy_timeout=5000")
    conn.execute("PRAGMA synchronous=NORMAL")
    conn.row_factory = sqlite3.Row
    return conn
```
This allows concurrent reads + writes without blocking.

### Shared httpx Client
In `scraper.py`, reuse a single `httpx.AsyncClient` instead of creating one per request:
```python
class FoodScraper:
    def __init__(self):
        self.client = httpx.AsyncClient(timeout=self.timeout, headers=self.headers)
    
    async def close(self):
        await self.client.aclose()
```

### Skip Already-Scraped Foods
In `bulk_scraper.py`, check DB before scraping:
```python
# Before scraping, check if food already has carbs in DB
if self.db.food_exists(food_id):
    continue  # skip, already scraped
```

### Database Singleton
Change `get_database()` to return a singleton instead of new instance per call:
```python
_db_instance = None
def get_database():
    global _db_instance
    if _db_instance is None:
        _db_instance = SQLiteDatabase()
    return _db_instance
```

## Feature 4: Smart Scraping Priority (Pause Fan-Out)

### Mechanism
Add an `asyncio.Event` to BackgroundScraper that user requests can signal:

```python
class BackgroundScraper:
    def __init__(self):
        self._user_request_active = asyncio.Event()
        self._user_request_active.set()  # scraping allowed by default

    async def pause_for_user(self):
        """Called by user request handlers - blocks scraping fan-out."""
        self._user_request_active.clear()
        # Wait for any in-flight fan-out to complete
        await asyncio.sleep(0.1)

    def resume_after_user(self):
        """Called after user request completes."""
        self._user_request_active.set()
```

### In BackgroundScraper._scrape_batch:
```python
async def _scrape_batch(self, batch):
    # Wait if user request is being served
    await self._user_request_active.wait()
    tasks = [self._scrape_one(item) for item in batch]
    await asyncio.gather(*tasks)
```

### In main.py search routes:
```python
@app.get("/api/foods/search")
async def search_foods_get(q: str = Query(...)):
    await bg_scraper.pause_for_user()
    try:
        # ... existing search logic ...
    finally:
        bg_scraper.resume_after_user()
```
