# Implementation Plan: Multi-Device Load Balancing, DB Viewer, TS Frontend, Pause/Resume Scraping

## Summary
Add 5 features to the food_ai Python/FastAPI app: (1) multi-device load balancing with failover via ngrok, (2) database viewer (dashboard + API), (3) TypeScript frontend client, (4) per-request scraper pause/resume, (5) search caching for performance.

---

## Phase 1: Foundation (no dependencies)

### 1.1 Update `config.py`
Add new env vars:
- `NODE_MODE` (standalone|coordinator|worker)
- `COORDINATOR_URL`, `WORKER_NAME`, `WORKER_NGROK_URL`
- `HEALTH_CHECK_INTERVAL=15`, `HEALTH_CHECK_TIMEOUT=5`, `HEALTH_CHECK_FAILURE_THRESHOLD=3`
- `WORKER_HEARTBEAT_INTERVAL=30`
- `CACHE_TTL=3600`, `CACHE_MAX_MEMORY=1000`

### 1.2 Update `models.py`
Add Pydantic models: `WorkerRegistration`, `WorkerInfo`, `DBQueryRequest`, `CacheStats`

### 1.3 Add pause/resume to `BackgroundScraper` in `bulk_scraper.py`
- Add `_paused: bool` and `_pause_event: asyncio.Event` fields
- Add `pause()` and `resume()` async methods
- In `_run_loop()` between batches: if `_paused`, await `_pause_event`
- Add `POST /api/scraper/pause` and `POST /api/scraper/resume` endpoints in `main.py`

### 1.4 Create `pause_middleware.py`
- `ScraperPauseMiddleware`: intercepts `/api/foods/search` requests, calls `bg_scraper.pause()` before processing, `bg_scraper.resume()` after response sent
- Add to `main.py` middleware stack

### 2.1 Update `database/base.py`
Add abstract methods: `get_table_info()`, `get_table_data()`, `get_db_stats()`, `execute_readonly_query()`

Implement the 4 new methods:

## Phase 3: Search Cache

### 3.1 Create `cache.py`
`SearchCache` class:
- In-memory LRU `OrderedDict` (max 1000 items)
- SQLite `search_cache` table for persistence
- `get(query, lang)` → check memory first, then DB
- `set(query, lang, response)` → store in both layers
- `invalidate()` → clear cache
- `stats()` → hit rate, sizes

### 3.2 Create `search_cache` table
Add migration in `cache.py` init: `CREATE TABLE IF NOT EXISTS search_cache (query_hash TEXT PRIMARY KEY, query TEXT, lang TEXT, response_json TEXT, created_at TEXT, hit_count INTEGER DEFAULT 1)`

### 3.3 Update `main.py`
- Wrap search endpoints with cache check: `cached = await cache.get(q, lang)` before DB lookup
- Store results in cache after search completes
- Add `GET /api/cache/stats` and `POST /api/cache/invalidate` endpoints

## Phase 4: Load Balancing & Worker System

### 4.1 Create `coordinator.py`
`Coordinator` class:
- `register_worker(name, url)` → save to DB `workers` table
- `unregister_worker(worker_id)` → remove
- `heartbeat(worker_id)` → update last_heartbeat
- `health_check_all()` → background task every 15s, GET each worker's `/api/health` with 5s timeout, 3 failures → mark down
- `select_worker(strategy)` → round-robin skip down workers, fallback to self
- `forward_request(method, path, body)` → pick worker, pause its scraper, forward request, resume scraper, return result
- `get_worker_status()` → list all workers with status

### 4.2 Create `workers` table
```sql
CREATE TABLE IF NOT EXISTS workers (
    worker_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    url TEXT NOT NULL,
    registered_at TEXT NOT NULL,
    last_heartbeat TEXT,
    status TEXT DEFAULT 'active',
    current_load INTEGER DEFAULT 0
);
```

### 4.3 Create `worker.py`
`WorkerClient` class:
- `register()` → POST to coordinator `/api/workers/register`
- `heartbeat_loop()` → background task sending heartbeat every 30s
- `unregister()` → POST to coordinator `/api/workers/unregister/{id}`

### 4.4 Update `main.py`
- Add worker management endpoints: `POST /api/workers/register`, `POST /api/workers/unregister/{id}`, `GET /api/workers`, `POST /api/workers/heartbeat/{id}`, `POST /api/workers/pause-all`, `POST /api/workers/resume-all`
- Add `worker` CLI command: `python main.py worker --coordinator <url> --name <name> --ngrok <url>`
- In lifespan: if `NODE_MODE=coordinator`, start health check loop; if `NODE_MODE=worker`, register with coordinator + start heartbeat

### 6.2 Lifespan changes
```python
async def lifespan(application):
    init_cache()  # create search_cache table
    if config.NODE_MODE == "coordinator":
        coordinator = Coordinator()
        coordinator.start_health_checks()
    elif config.NODE_MODE == "worker":
        worker = WorkerClient(coordinator_url, self_url, name)
        await worker.register()
        worker.start_heartbeat()
    if config.BACKGROUND_SCRAPER_ENABLED:
        await bg_scraper.start()
    yield
    if config.NODE_MODE == "worker":
        await worker.unregister()
    await bg_scraper.stop()
```
