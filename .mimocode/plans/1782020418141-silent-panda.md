# Plan: Fix CORS Error & Ensure Scraper Reliability

## Problem
1. **CORS Error**: API requests get "Disallowed CORS origin" because `main.py:121` only allows `https://localhost` and `http://localhost`.
2. **Scraper reliability**: User wants the scraper to always work (no special conditions).

## Changes

### 1. Fix CORS in `main.py` (lines 119-125)

**Current code:**
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://localhost", "http://localhost"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**New code:**
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

This allows any origin. FastAPI's `CORSMiddleware` handles `allow_origins=["*"]` + `allow_credentials=True` by reflecting the request's `Origin` header back (browsers reject `Access-Control-Allow-Origin: *` when credentials are present, so it echoes the caller's origin instead).

### 2. No scraper changes needed

The scraper already works with multi-level fallback (cache → DB → 11 web sources → 6 AI providers). All sources have try/except handling that silently skips failures. The `scrape_all()` method fans out to all sources concurrently. No code changes required.
